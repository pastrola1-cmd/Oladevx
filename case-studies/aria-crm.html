Multi-Tenant SaaS CRM

ARIA CRM Platform Architecture

Architected and deployed a multi-tenant CRM for international education agencies to automate admissions, track student pipelines, and audit application files using Claude API.

Client / Organization Global Education Consultancies
Role Lead Systems Architect
Tech Stack Next.js 14, Supabase RLS, Claude API, Resend, Vercel
Deployment Year 2026

The Business Problem

International education agencies process thousands of student visa and university applications every intake. Working across multiple countries, agencies face difficulties coordinating workflows, protecting student personal data, and detecting document fraud or missing credentials in student folders.

Legacy tools lacked multi-tenant data isolation and did not offer automated validation for academic transcripts or statements of purpose, requiring hours of manual review.

The Solution: ARIA CRM

Under the OLADEVVX banner, we designed and built ARIA CRM, a modern multi-tenant software system. The platform allows agencies to instantiate their own isolated pipelines, assign tasks to sub-agents, and track student application progress in real-time.

We integrated the Claude API to analyze academic transcripts and critique statements of purpose, automatically flagging verification gaps before files are submitted to universities.

Technical Architecture

Database Isolation & AI Audits

  • Supabase RLS Policies: Multi-tenant security ensuring tenant data separation at the database level.
  • Claude API Integration: Automated LLM pipelines evaluating student transcripts and suggesting improvements for essays.
  • Resend Email Engine: Automated notification loops notifying students immediately upon document acceptance or rejection.
  • Next.js 14 App Router: Server-rendered pipelines with sub-second page transitions.
